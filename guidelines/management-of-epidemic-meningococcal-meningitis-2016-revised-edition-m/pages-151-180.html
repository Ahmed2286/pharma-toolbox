Appendix 15. Estimating needs: meningitis treatment in an epidemic context

Excel File 2. “Estimating needs: meningitis treatment in an epidemic context” is used to
calculate treatment needs according to the epidemiological data. It facilitates the placing
of orders and the drawing up of a provisional budget. The first order should cover
treatment needs for a reasonable period, keeping in mind the time taken for the order
process and reception of delivery (4 to 8 weeks). Following orders depend upon the
progress of the epidemic and subsequent needs.

Use of sheets

Use one Excel file for each district. Each file includes:

– One sheet listing the composition of treatment kits
– 4 “Estimation of meningitis treatment” sheets. If necessary, copy and paste to add
new sheets.
– Only fill in the yellow cells.

“Estimation of meningitis treatment” sheets

Use one sheet for each order. Record

– The expected attack rate for the epidemic period: this is difficult to predict. If data
are not available for the district, choose an incidence of 150-300,000 per 100,000
population.
– The buffer stock required: generally 25%. The buffer may be set at a higher level for
the first order to prevent stock ruptures.
– The proportion of hospitalised cases expected: generally 15-20%, but the proportion
depends on the access to healthcare and the strategy of decentralised care.
– The name of the district.
– The total population.
– The number of cases already reported.
– The unit price of each item in the kits.

The file automatically calculates

– The total number of cases estimated for the epidemic and the number of cases yet
to present.
– The quantity of medication required for the buffer stock
– The quantity of medication required to manage the number of expected cases
– Treatment requirements for simple and hospitalised cases
– The quantity of medication required to make up the treatment kits
– The estimated cost in Euros

In the “other items” table, record useful items not included in the kits, which can be
supplied with the first donation.

Appendix 17. Tracking availability of meningitis treatment

Excel File 3. “Tracking availability of meningitis treatment” is used to monitor the
availability of treatment by district and by health zone and to identify the sites where
rapid supply is needed.

Use of sheets

Use an Excel file for each region. Each file includes:

– 8 “District” sheets – copy and paste to add new sheets if needed
– Only fill in the yellow cells

“District” sheets

At the time of first use

– Record the year, name of the region, name of the district and the total population
for each district
– Record the source of the information
– Record the number of the epidemiological week which marks the start of the
supply of treatment
– Treatment of uncomplicated cases: record the name of each zone, the population,
number of cases and deaths and the number of treatments supplied. The CFR and
number of treatments available are calculated automatically.
– Treatment of hospitalised cases: record the data for the reference hospital.
– Record the name of the district on the tab.

Each week

– Record the number of the epidemiological week
– Record the number of cases and deaths for each zone and hospital using data from
the epidemiological reporting sheets. If there are no cases, record “0”. An empty
cell does not mean there are no cases, but that data are incomplete.
– Record the number of treatments distributed using the donation certificates. Record
“0” if there has not been any supply during the week.

Weekly analysis of data

– If the CFR is high, make a visit to the health zone
– The number of treatments available should be equal to or higher than the number
of estimated cases expected for the coming week or period before the next supply.
– Estimate the number of treatments required according to the epidemiological
curve, any vaccination performed or planned and the frequency of supply.
– Record the analysis and intended action in the box for weekly analysis.

Appendix 25. Monitor card Stop!Watch®with Freeze tag®

Card with temperature indicators, used to monitor the temperature inside a
refrigerator.
This card provides an additional temperature check, but it is still essential to check the
temperature of the refrigerators twice daily.

Specifications
Double-sided card, bilingual English/French, with 2 indicators:

– A heat-sensitive indicator with 4 windows (A, B, C, D), that gradually turn blue
depending on the length of time and the temperature they are exposed to:
 Windows A, B and C turn blue when the indicator is exposed to
temperatures above 10°C.
 Window D turns blue when the indicator is exposed for 2 hours to
temperatures above 34°C.
– A freezing indicator (Freeze-tag®): when the indicator is exposed to temperatures
below 0°C for more than one hour, the display will change from the “OK” setting to
the “ALARM” setting.

Instructions for use

– Place the card in the refrigerator 30 minutes (+2°C to +8°C) before activating it.
– Then activate the card by removing activation tab.
– The Stop!Watch® card has to stay permanently in the refrigerator and must be
checked each time the twice daily temperatures are checked.
– If the windows are blue or if the Freeze tag® indicates “alarm”:
 Note the change on the card and place a new card into therefrigerator.
 Inform the supervisor who will decide on the appropriate action to be taken.

Each Excel file includes:

– 3 example sheets: vaccination schedule by site, vaccination schedule by team,
estimating freezing capacity.
– 3 “Estimating freezing by site” sheets named A to C and 1 supplementary sheet
with space to record 40 vaccination sites.
– Use one sheet per zone or district. If needed, cut and paste to add new sheets.
– Only fill in the yellow cells.

For each sheet

1. Establish the cold chain equipment for a vaccination site

– One site can accommodate a maximum of 2 vaccination teams.
– One cool box is required for each site and one vaccine carrier for each team.
– Select the appropriate option in the yellow boxes to record the ambient
temperature and the frequency of rotation of ice packs.
The file automatically calculates in Table A:

– The number of cool boxes and vaccine carriers per team
– The number of ice packs per cool box and vaccine carrier
– The total number of ice packs at day 1
The file automatically calculates in Table B:

– The total number of ice packs needed each day according to the frequency of
rotation and number of teams.

2. Estimate the number of ice packs needed each day for the total number of teams and
sites.
In Table C:

– Record the name of the district or zone and the name of the vaccination sites
– For each site, record the number of teams and the name or number according to the
vaccination plan.
– Then fill in the number of ice packs needed each day according to the calculation
given in Table B.
– The file automatically calculates the total daily ice pack needs.

– In Table D, fill in the number of available working freezers according to the

The file automatically calculates

– The total freezing capacity available
– The total ice pack storage capacity
– The total freezing capacity per day
– The number of days needed to ensure that all ice packs are frozen on day 1

4. Change in daily ice pack needs for vaccination sites
In the box for day 1, fill in the maximum number of ice packs stored or the maximum

The file automatically calculates:

– Exits for each day (= total number of ice packs handed out each day to vaccination

– Balance (= number of ice packs available each day).
