MONTHLY PHARMACY ANALYSIS REPORT

HOW TO FILL IT IN
(Version January 2024)

INTRODUCTION

The report is needed to:

•
Help the pharmacy responsible to structure his analysis and make existing management problems visible to take appropriated decisions.
•
Help the coordination to analyze the pharmacy management, be alerted on time and follow the project activity.
•
Help the HQ to do a macro analysis, improve the support and capitalize the information.
•
Have a written proof of the pharmacy activity in the project – it may be requested in case of an external audit.

For these reasons, the monthly pharmacy analysis report is mandatory.

It must be filled in by the person in charge of the project pharmacy and must be sent with the last backup from Isystock, (with the last GH-TSR , The expiry tool and the
valued inventory report ) to the CoPharma/Country Pharma with the MedCo in copy – latest the 07th of the month.

The CoPharma/Country Pharma (or MedCo if the positions are not provided) sent the report and the backup, (with the last GH-TSR , The expiry tool and the valued
inventory report ) to the MedCell & parispharma@paris.msf.org – latest the 10th of the month.

PREREQUISITES

Having an updated Isystock that means:

•
Prepare the GH-TSR for the month of the report.
•
All movements of the last month are recorded; ensure that there is no negative value in the initial stock and/or the theoretical stock.
•
The inventory (complete or partial) is recorded.

IMPORTANT CONCEPTS TO CONSIDERE

-
Project stock managed at coordination level: this stock should be taken in consideration. The coordination team must send at the end of the month the situation
of project stock managed at their level.

-
Update attributes: "consumable" and "non-consumable" attributes in Isystock are for information only, it is advisable for each project to adapt these attributes to
the realities of the field.

-
Analysis Method: different approach of analysis can be considered.

o Using AMC help to have a clear idea of what has been done and report a picture of pharmacy statement.
o Using FMC help to do deep analysis on what will be the stock situation in the future and to take decision for donation. It needs a good and update

forecast.

For the monthly report we advise using AMC.

PROJECT INFORMATION SHEET

General project pharmacy related information which tends to be stable, thus there is no need to update it monthly, only when there are changes in that information.

PROJECT CODE and ISYSTOCK CODE
The Isystock code identifies the project pharmacy and is composed of the project code (the two ID letters of the country + the figures of the budget code) plus three
letters.
Example CD146RUT = CD (DRC) + 146 (Rutshuru project) + RUT (Rutshuru pharmacy)

The code is visible in the headline of the Isystock.
If the code does not correspond to the mentioned rule, contact your Isystock referent in Paris for an update.

CURRENCY USED
All article prices must be in the same currency, in general in Euro. Depending on the country, another currency can be used after discussion with the Isystock referent in
Paris and the financial department.

STOCK VALUE AT THE END OF THE PREVOUIS YEAR
The value at the end of previous year will be used to make a matching of all movement at the end of the current year.

HUMAN RESSOURCES
A brief description on the human resources working on the pharmacy: they level of experience / seniority, trainings, etc.

GOOD STORAGE PRACTICES (GSP)
Good storage practices have a direct impact on medical items quality. It is important to identify if they are correctly in place, and to define action points if need it. Wrong
GSP have a negative impact on stock management.
Select “YES” or “NO” depending on the status in the storage area and indicate the complementary information when requested.
Storage capacity is important to respect GSP. If a clear storage capacity/need is not available, you can contact the pharma referent who will in collaboration with logistic
and supply departments provide the support needed.

QUARTERLY/MONTHLY NOTION

The monthly report is now composed of 2 main sections: a section that is to be filled on a quarterly basis and a section that is to be filled monthly. To be able to use this
option please to activate the content after opening the file.

-
Quarterly chapters: it is advisable to complete these chapters quarterly, but if a project wish to fill them monthly, they still can. Quarterly data must consider all
movements since the last analysis. Example: if the last analysis was done in July, and now you are doing one in October, the one of October must consider the IN /
OUT movements, the losses and all other the movements from July to October. Quarterly analysis of the Month of December is always mandatory, because it is
used to complete the annual report.

-
Monthly chapters: no change compared to other years.

THEORETICAL STOCKS AND PHYSICAL STOCK (INVENTORY)

Each month an inventory must be done (complete or partial – see procedure « inventory »).

INVENTORY
Complete inventory: All items are counted.
Partial inventory: At least 10% of the number of referenced items is counted.

COMPARISON OF THEORETICAL STOCKS (STOCK CARDS & ISYSTOCK)
Each month a comparison of the stock must be done between stock cards and Isystock before the monthly inventory (at least for the inventoried items).
The first time this will be heavy and time-consuming work, but it will save time and make the inventory more fluid in the future.
For items with a difference, the movements need to be compared and you need to search for an explication.

Analysis

The different sources of error need to be detailed; the percentage linked to the total number of differences will be calculated automatically. Like that, you can take
appropriated measures to improve the next inventory.
